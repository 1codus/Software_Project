# ✈️ 여행 예약 시스템 (Travel Reservation System)

> **동의대학교 소프트웨어설계공학 프로젝트 (2분반 4팀)**
> 직관적인 지도 기반 인터페이스를 통해 여행 계획부터 결제까지 원스톱으로 해결하고, 사업자와 관리자에게는 효율적인 관리 도구를 제공하는 통합 예약 플랫폼입니다.
> 
> 

---

## 📌 서비스 개요 (Vision)

* **사용자:** 지도에서 지역을 선택하는 직관적인 검색부터 결제, 예약 조회 및 변경까지 신속하게 수행.


* **사업자:** 호텔, 항공, 렌터카 상품을 간편하게 등록하고 실시간으로 예약을 관리.


* **관리자:** 회원 및 상품 관리, 예약 상태 모니터링을 통해 시스템 운영 효율성 극대화.



---

## 🛠 기술 스택 (Tech Stack)

* **Language:** Java
* **GUI:** Java Swing (GUI 개발 및 지도 이미지 처리) 


* **Data Management:** Java 파일 입출력(File I/O), 객체 직렬화 


* **Build Tool:** Maven (NetBeans IDE) 


* **Version Control:** Git (분산 저장소 방식) 



---

## 🏗 디자인 패턴 (Design Patterns)

본 프로젝트는 확장성과 유지보수성을 위해 다양한 객체지향 디자인 패턴을 적용했습니다.

| 패턴 | 적용 내용 |
| --- | --- |
| **MVC** | UI(View), 비즈니스 로직(Controller), 데이터 관리(Model) 분리 |
| **Factory Method** | 상품 유형(호텔, 항공, 렌터카)에 따른 유연한 객체 생성 |
| **Builder** | 복잡한 예약 정보의 단계별 생성 및 선택적 옵션 추가 |
| **Strategy** | 사용자/사업자 모드에 따른 동적 기능(알고리즘) 전환 |
| **Observer** | 신규 상품 등록 등 상태 변경 시 관리자에게 실시간 알림 |
| **Singleton** | 설정 정보 및 DB 연결 인스턴스의 유일성 보장 |
| **Decorator** | 조식 추가, 차량 옵션 등 상품 서비스의 동적 확장 |
| **State** | 예약 상태(결제 완료, 사용 완료 등)에 따른 객체 행동 제어 |

## 🏃‍♂️ 개발 프로세스 (Development Process)

**애자일 방법론(Scrum)**을 채택하여 지속적인 개선과 협업을 실천했습니다.

* **스프린트:** 1주일 단위로 목표 설정 및 작업 수행.


* **데일리 스크럼:** 매일 Notion을 활용한 진행 상황 공유 및 이슈 해결.


* **회고 및 리뷰:** 매주 화요일 정기 회의를 통해 성과 검토 및 프로세스 개선.

## 🔍 주요 기능 (Key Features)

* **지도 기반 검색:** 자바 스윙을 이용한 GUI로 목적지 시각화 및 선택.


* **통합 예약 시스템:** 호텔, 항공, 렌터카 정보를 파일 시스템으로 안전하게 관리.


* **예외 처리:** 안정적인 복구와 사용자 피드백을 위한 로깅 및 예외 처리 로직 적용.
